Method Overloading

Method overloading means defining multiple methods in the same class with the same name but different parameters (type, number, or order).

Purpose: To perform different tasks with the same method name, improving code readability.

Key Points

  • Methods can have any name, not necessarily the class name.
  • Return type can be the same or different, but does not affect overloading.
  • Java decides which method to run based on the method signature.

Constructor Overloading

Constructor overloading means defining multiple constructors in a class with different parameter lists.

Purpose: To allow objects to be initialized in different ways.

Key Points

  • Constructors have no return type.
  • They must have the same name as the class.
  • Java selects the appropriate constructor based on the arguments passed when an object is created.
